Technické specifikace

Model MicroBlink v1 s velikostí 3 miliony parametrů reprezentuje špičku v oblasti tzv. edge-computing modelů. Byl vyvinut za účelem extrémně levného a rychlého nasazení.

Parametr Hodnota / Specifikace
Celkový počet parametrů 3 240 500 (~3M)
Architektura základního LLM Causal Decoder-Only Transformer (upravené vrstvy)
Maximální délka kontextu 512 tokenů (ideální pro věcné Q&A)
Hlavní funkce a filtry Identity Guard v1, Striktní RAG cross-check
Doporučená konfigurace hardware CPU (libovolné moderní), CUDA (volitelně - spotřeba <100MB VRAM)

Architektura a trénovací proces

MicroBlink_v1 byl trénován metodou selektivní destilace znalostí z větších jazykových modelů a následně laděn na specifických datech českého internetového prostoru. Primární důraz byl kladen na schopnost odolat halucinacím – pokud model nedostane v textovém kontextu přesná fakta, zasáhne křížová kontrola slov a model odmítne odpověď fabulovat.

  • Tokenizace: Používá upravený BPE (Byte-Pair Encoding) slovník s plnou nativní podporou české diakritiky bez penalizace délky kontextu.
  • Smart Switch: Model interně obsahuje rychlou matematickou a faktickou rozhodovací vrstvu, která jednoduché encyklopedické definice filtruje okamžitě, bez nutnosti generování celého grafu.

Příklad lokální inference (Python)

Díky minimalistickým rozměrům lze model zavolat v produkčním skriptu pomocí knihovny `transformers` v několika řádcích kódu:

import torch
from transformers import AutoTokenizer, AutoModelForCausalLM

# Načtení z lokálního úložiště
tokenizer = AutoTokenizer.from_pretrained("./muj_tokenizer")
model = AutoModelForCausalLM.from_pretrained("./muj_3m_model").to("cpu")

prompt = "Kontext: Elon Musk je podnikatel.\nOtázka: Kdo je Elon Musk?\nPodrobná odpověď:"
input_ids = tokenizer.encode(prompt, return_tensors="pt")

with torch.no_grad():
    output = model.generate(input_ids, max_new_tokens=50, do_sample=False)

print(tokenizer.decode(output[0], skip_special_tokens=True))
